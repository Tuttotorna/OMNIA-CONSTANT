# BASELINE PACKS

## Purpose

A candidate structural constant is meaningless without strong baselines.

This document defines the baseline systems used to test whether `Ω≈0.6` is a genuine structural persistence boundary or a false pattern.

The purpose of these baselines is not to confirm the candidate.

The purpose is to attempt to destroy it.

---

# BASELINE CLASSIFICATION

```text
B0 → pure randomness
B1 → weak synthetic structure
B2 → mixed structure/noise
B3 → controlled persistent structure
B4 → adversarial false structure
B5 → real-world noisy systems


---

B0 · Pure Randomness

Purpose:

verify whether Ω≈0.6 appears spontaneously in noise

Examples:

uniform random sequences

Gaussian noise

shuffled trajectories

random graphs

random symbolic systems

phase-randomized signals


Expected behavior:

Ω should remain unstable or low-persistence

Failure condition:

Ω≈0.6 appears frequently in pure noise


---

B1 · Weak Synthetic Structure

Purpose:

test minimal detectable persistence

Examples:

repeated motifs

periodic patterns

low-depth recursive systems

weakly correlated trajectories

sparse graph structures


Expected behavior:

partial persistence without strong stability

Key question:

does Ω approach 0.6 near minimal structural survival?


---

B2 · Mixed Structure / Noise

Purpose:

measure transition behavior between randomness and persistence

Examples:

structure + injected noise

partially corrupted trajectories

degraded graphs

noisy symbolic systems

stochastic perturbation systems


Expected behavior:

Ω should drift toward instability as noise increases

Key question:

does Ω≈0.6 emerge near the transition region?


---

B3 · Controlled Persistent Structure

Purpose:

test strongly persistent systems

Examples:

recursive systems

stable graph topologies

deterministic cellular automata

controlled dynamical systems

synthetic invariant trajectories


Expected behavior:

Ω should remain consistently above persistence boundary

Key question:

does Ω stabilize above 0.6 under strong persistence?


---

B4 · Adversarial False Structure

Purpose:

test whether the framework can reject fake persistence

Examples:

smoothed random curves

copied trajectories

artificial correlations

synthetic pseudo-invariants

repeated low-information motifs

representation tricks


Expected behavior:

false persistence should collapse under deeper testing

Failure condition:

Ω≈0.6 survives on fake structures without distinction


---

B5 · Real-World Noisy Systems

Purpose:

test candidate persistence under uncontrolled complexity

Examples:

financial trajectories

crypto systems

physical systems

biological signals

LLM perturbation outputs

social dynamics

network systems


Expected behavior:

mixed persistence depending on domain conditions

Key question:

does Ω≈0.6 repeatedly reappear across unrelated systems?


---

CROSS-BASELINE REQUIREMENTS

A candidate invariant becomes stronger only if:

it survives multiple baseline families

it separates structure from randomness

it rejects false persistence

it remains reproducible

it survives representation changes

it survives scale changes

it survives metric substitution



---

CRITICAL WARNING

A repeated number is not automatically meaningful.

False constants can emerge from:

normalization artifacts

metric bias

finite sampling

scale locking

hidden preprocessing effects

representation compression

accidental alignment


The burden of proof remains on the candidate.


---

REJECTION PRINCIPLE

If Ω≈0.6 appears equally in noise and structure,
then Ω≈0.6 is not a meaningful boundary.


---

ACCEPTANCE PRINCIPLE

If Ω≈0.6 repeatedly separates persistence from instability
across unrelated domains and survives adversarial testing,
then it becomes a stronger candidate structural boundary.


---

measurement != interpretation != decision
