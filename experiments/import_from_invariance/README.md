# Import From OMNIA-INVARIANCE

## Purpose

OMNIA-CONSTANT does not replace OMNIA-INVARIANCE.

It extends it.

OMNIA-INVARIANCE generates and compares structural trajectories across domains.

OMNIA-CONSTANT imports those trajectories and tests whether persistent invariant boundaries emerge from them.

---

## Core relationship

```text
OMNIA
→ structural measurement

OMNIA-INVARIANCE
→ trajectory-space analysis

OMNIA-CONSTANT
→ invariant persistence validation


---

Main objective

This experiment family exists to determine whether:

Ω≈0.6 behaves like a persistent structural boundary
across trajectory-space experiments generated by OMNIA-INVARIANCE.

Key question

Does Ω≈0.6 repeatedly emerge near the transition
between instability and persistence?


---

Important distinction

OMNIA-INVARIANCE studies trajectory geometry.

OMNIA-CONSTANT studies whether persistent invariant regions emerge from that geometry.

These are related but different goals.


---

Failure principle

If Ω≈0.6 disappears when importing across domains, representations, scales, or metrics, then the candidate weakens.

If Ω≈0.6 repeatedly reappears across independent trajectory families, then the candidate strengthens.


---

Core principle

measurement != interpretation != decision
