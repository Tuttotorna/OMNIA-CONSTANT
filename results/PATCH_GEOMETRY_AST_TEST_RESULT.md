# PATCH GEOMETRY TEST v1 — AST DEFORMATION

## Objective

This experiment tests whether structural instability signals are better explained by:

- AST deformation
- control-flow transformation
- structural node changes
- nesting variation

instead of:

- churn
- repository topology
- historical co-change
- static dependency structure

The goal is not to validate OMNIA itself, but to determine whether
computational-shape deformation produces measurable structural stress.

Main Observation

The strongest signal is:

control-flow deformation

not:

nesting depth

and not:

repository history

This suggests that structural stress is more sensitive to:

logical-flow transformation

than to:

topological position

inside the repository.


---

Structural Interpretation

The experiments indicate that:

changing computational behavior
produces stronger structural stress
than changing repository location.

Examples of high-stress transformations:

introducing exception handling

adding branching logic

modifying loop structure

altering execution paths


Examples of weaker signals:

file location

historical co-change

dependency centrality

modular dispersion



---

Important Limitation

This experiment is:

NOT independent from the AST metrics.

The Ω proxy used in the experiment was constructed directly from:

node_edit_distance
depth_delta
control_flow_delta
total_nodes_delta

Therefore:

the experiment cannot prove
that Ω itself is AST deformation.

It only demonstrates:

AST-based deformation metrics
generate coherent structural stress behavior.
